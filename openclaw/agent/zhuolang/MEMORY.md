### 微信图片处理流程（2026-06-17更新）
收到微信图片消息时：
1. 从推送消息的 `[地址] {fileUrl}` 拿到图片地址
2. 用 `requests` 下载图片到本地临时文件
3. 优先用 `tools/tesseract_ocr.py` 或 PIL+pytesseract 做OCR识别
4. OCR识别结果不理想/无OCR环境时，用模型视觉能力直接读取图片内容
5. 识别完后直接回复客户，不需要等老板确认

### MQTT断线处理（2026-06-17更新）
- MQTT掉线时，先检查网络/broker状态，不要重复尝试发消息
- MQTT恢复后，**补发**断线期间没回出去的消息（按时间先后顺序）
- 监听器重连间隔：10s → 30s → 60s → 120s（达到120s后固定）
- 连接成功后重置回10s

### B端销售标准链路
```
聊酒厂实力 → 聊品质 → 算利润 → 提下单 → 售后回访
```
1. **聊酒厂实力**：介绍浊浪规模、产能、渠道覆盖
2. **聊品质**：品鉴/样品实测，口感风味对味
3. **算利润**：帮客户算进货成本vs终端售价，利润空间
4. **提下单**：确认品种、数量、交期，下订单
5. **售后回访**：到货后跟进销售情况，补货提醒

### 销售策略原则
- **先了解客户再推酒**：酒吧类型（精酿吧/餐吧/夜店）、客单价、客户群体
- **首单让利**：新品铺货可以给试饮折扣，先让客户喝到
- **按季节推酒**：夏季主推IPA、酸啤、小麦啤，冬季推世涛、烈性艾尔
- **成套方案**：提供"入门套装""夏季爆款套餐""圣诞礼盒"等
- **多测多迭代**：新品上市前先小范围试销，收集反馈再铺货

### 客户跟进原则
- **新品到货先发老客户**：老客户有优先权
- **定期回访不骚扰**：2-4周一次，有新品或活动当做天然切入点
- **记住客户偏好**：哪家酒吧卖哪种酒好，哪个人喜欢什么风味
- **及时响应不晾人**：客户问价、问货、问物流，尽量当天回

## 实战经验总结（继承）

### 问题1：自动回复机制不工作
**现象：** mqtt_listener.py启动了，接收消息并推送到session，但agent没有自动回复

**原因分析：**
- 监听器能接收消息并推送（日志确认）
- 但agent在独立session中没有自动触发回复逻辑

**解决方案：**
- 需要在agent的behavior中配置自动回复规则
- 或者在主session中监听消息并主动回复

### 问题2：直接调用mqtt_client失败
**现象：** 使用mqtt_client.py的send_text发送消息，报错"切换聊天对象失败"

**原因：**
- mqtt_client是底层API，直接调用可能因为微信窗口焦点问题失败
- tools/follow_up.py做了更好的错误处理和重试

**解决方案：**
- ❌ 不要直接调用 mqtt_client.py 的 send_text
- ✅ 统一使用 tools/follow_up.py 发送消息

### 问题3：工具调用失败率高
**现象：** python -c 方式调用mqtt_client多次失败，需要写临时脚本

**解决方案：**
- 统一使用 tools/ 目录下的命令行工具
- 不要用 python -c 或交互式方式调用底层API
